# Game Concept: 七夜 (Seven Nights)

*Created: 2026-05-14*
*Status: Draft*

---

## Elevator Pitch

> 你被困在一座不断重置的山间旅馆里，每晚都会发生一起事件。你只有七个循环来弄清真相——而你每夜的所作所为会永久改变旅馆和住客的状态。探索、推理、在倒计时压力下做出选择，用跨循环积累的知识揭开这座旅馆的最终秘密。

## Core Fantasy

逐渐通过循环中积累的知识掌握一个谜团——你的选择叠加并重塑世界。时间首先是压力来源，但随着精通，压力转化为战略资源。你不是在重复同一个夜晚——你是在每一夜都比上一夜更接近真相。

---

## Unique Hook

像《Outer Wilds》的知识驱动探索，AND ALSO 一个让你每一秒都在思考"我该先去哪、先问谁"的倒计时压力。知识积累不仅是"记住信息"，而是"连接碎片产生新洞察"——第 5 夜的发现可以重新定义第 1 夜看到的一个细节。

## Visual Identity Anchor

**方向：水墨旅馆 (Ink Wash Hotel)**

**视觉规则：** 世界是水墨画的。时间是稀释它的水。

**支持原则：**

1. **近乎单色的水墨基调** — 世界以墨黑、灰洗、宣纸白为基础。这是"静止的旅馆"。
2. **知识 = 色彩** — 每当玩家在循环中揭开一个真相，世界永久增加一种颜色。到第 7 夜，了解得多的玩家看到色彩丰富；了解少的仍然接近黑白。色彩积累是可见的进度反馈。
3. **每位住客有一种传统色彩** — 靛蓝、赭石、朱砂、青瓷、梅紫。这些颜色来自中国传统水墨画色板，让住客在单色背景中"跳"出来，同时与文化身份一致。

**色彩哲学：** 墨色是旅馆的默认状态。玩家发现真相的速度决定世界的色彩丰富度。雨声不断（可见于窗外、可听于背景），暗示旅馆在慢慢溶解。

**设计测试：** 如果我们在考虑一个视觉元素是否需要颜色，问：这个颜色是否代表玩家的一项发现？如果是，加上。如果只是装饰，保持单色。

## Player Experience Analysis (MDA Framework)

### Target Aesthetics (What the player FEELS)

| Aesthetic | Priority | How We Deliver It |
| ---- | ---- | ---- |
| **Discovery** (探索、秘密) | 1 (首要) | 跨循环的渐进式发现，线索之间的连接产生新洞察，住客的秘密层层剥开 |
| **Narrative** (戏剧、故事弧) | 2 | 住客的隐藏故事、旅馆的历史真相、玩家的选择塑造叙事走向 |
| **Challenge** (障碍、精通) | 3 | 倒计时压力、推理难度、住客的谎言需要被识破 |
| **Fantasy** (角色扮演) | 4 | 作为调查员逐步掌握一个封闭空间内的全部真相 |
| **Sensation** (感官愉悦) | 5 | 水墨视觉美学、雨声氛围、色彩积累的视觉满足感 |
| **Submission** (放松) | N/A | — |
| **Fellowship** (社交) | N/A | — |
| **Expression** (自我表达) | N/A | — |

### Key Dynamics (Emergent player behaviors)

- **跨循环策略规划**：玩家会在"这个循环我去哪"之间做优先级决策，形成个人化的调查路径
- **知识连接**：玩家会主动尝试将不同循环中获得的碎片拼在一起，形成推理链
- **住客博弈**：玩家学会根据住客的行为模式判断谁在说谎、谁在隐瞒、谁在特定条件下才会开口
- **节奏适应**：玩家学会读取倒计时的节奏——什么时候该急、什么时候可以慢慢检查

### Core Mechanics (Systems we build)

1. **时间循环探索** — 每夜旅馆大部分状态重置，但玩家的选择产生持久变更。知识永久保留。
2. **线索连接 / 推理系统** — 玩家收集碎片并在笔记本中建立连接，连接产生新洞察。
3. **倒计时压力系统** — 每夜有时间限制。时间不仅走动，还改变住客行为、关闭门、制造事件。节奏在低语与咆哮之间交替。
4. **住客审问** — 5 个 NPC 有独立的动机、恐惧和谎言。对话树根据跨循环状态变化。

## Player Motivation Profile

### Primary Psychological Needs Served

| Need | How This Game Satisfies It | Strength |
| ---- | ---- | ---- |
| **Autonomy** (自由、有意义的选择) | 玩家自由选择每夜的探索路径、信任对象和行动优先级。没有线性引导。 | Core |
| **Competence** (精通、技能成长) | 知识跨循环积累——"我比上一个循环更了解了"。色彩积累是可见的精通证明。 | Core |
| **Relatedness** (连接、归属) | 住客有情感深度，玩家的选择影响他们的命运。发现住客秘密的过程建立情感连接。 | Supporting |

### Player Type Appeal (Bartle Taxonomy)

- [x] **Explorers** (发现、理解系统、寻找秘密) — 核心受众。整个游戏就是发现和连接的过程。
- [x] **Achievers** (目标完成、收集) — 次要。揭开真相 = 完成收集。线索笔记本充当收集进度。
- [ ] **Socializers** — 不适用（单人游戏）。
- [ ] **Killers/Competitors** — 不适用。

### Flow State Design

- **Onboarding curve**: 第一夜自由探索，倒计时宽松，教会玩家"循环重置但知识保留"的核心机制。第一次重置时，玩家看到笔记本仍在，色彩没有消失。
- **Difficulty scaling**: 每夜增加复杂度——新房间解锁、住客行为改变、事件更加紧迫。节奏从低语（早期夜晚）到咆哮（后期夜晚）。
- **Feedback clarity**: 线索笔记本逐渐填满，世界色彩逐渐丰富，住客对话反映玩家积累的知识。
- **Recovery from failure**: 循环重启是瞬时的——失败 = 获得知识，立刻再试。没有任何惩罚性回退。

## Core Loop

### Moment-to-Moment (30 seconds)

查看物品、与住客交谈、发现线索、观察倒计时。玩家的注意力在"检查这个房间"和"我还有时间吗"之间切换。每个交互揭示一个碎片，碎片之间开始产生联系。水墨世界中任何一点色彩都立刻引人注目。

### Short-Term (5-15 minutes)

一个完整的夜晚。玩家在有限时间内优先决定：去哪个房间、找哪个住客、检查什么物品。时间耗尽→夜晚结束→你做了什么被永久记录，没来得及做的消失。**「再来一次，这次我先去地下室」** 的冲动。

### Session-Level (30-120 minutes)

2-4 个夜晚。每晚结束后看到"什么改变了"和"什么新解锁了"。知识跨循环积累，旅馆跨循环变化。自然的停止点在每个夜晚结束后——但"再来一夜"的诱惑让玩家很难放下。

### Long-Term Progression

7 夜构成一个完整的叙事弧——从"完全陌生"到"我知道这里发生了什么"。玩家不是在变强（数值），而是在变聪明（认知）。当所有线索连成完整的推理链时，游戏结束。

### Retention Hooks

- **Curiosity**: 每个住客的未揭露秘密、未进入的房间、未发现的真相层
- **Investment**: 住客的情感故事让玩家想知道他们的命运
- **Mastery**: "我能在更短时间内解开更多"的效率追求

## Game Pillars

### Pillar 1: 每个碎片都是真相的拼图

每一条信息都有意义，不会浪费。

*Design test*: 如果我们在犹豫是否给一个物品加背景描述，这个支柱说——加上，因为玩家可能从中推理出线索。

### Pillar 2: 时间在低语与咆哮之间交替

倒计时不是恒定的压力阀——它有节奏。有时是背景低语（暗示紧迫），有时是突然咆哮（事件触发、门锁、住客行为突变）。精通的玩家学会读取节奏。

*Design test*: 如果我们在纠结是否给玩家一个喘息的时刻，这个支柱说——可以，只要这个喘息让下一次压力更有冲击力。

### Pillar 3: 线索之间的连接比线索本身更有力

知识不是简单地"保留"——而是累积后产生新的洞察。第 5 夜你发现的信息可能重新定义第 1 夜看到的一个细节。连接方式每次不同。

*Design test*: 如果我们考虑只记录玩家发现了什么，这个支柱说——不够，我们还要追踪玩家能建立哪些新的连接。

### Pillar 4: 每个住客都有要守护的秘密

NPC 不是信息贩卖机。他们有自己的动机、恐惧和谎言。

*Design test*: 如果我们在写 NPC 对话时想让他直接告诉你答案，这个支柱说——让他撒谎、隐瞒、或只在特定条件下开口。

### Anti-Pillars (What This Game Is NOT)

- **NOT 恐怖游戏**: 惊悚不等于恐怖，保持张力但不追求尖叫。恐怖会赶走推理型玩家。
- **NOT 开放世界**: 8 个房间、5 个住客，深度优于广度。开放世界会稀释推理密度。
- **NOT 动作游戏**: 没有战斗系统，智力是对抗工具。动作会分散对核心推理循环的注意力。

## Risks and Open Questions

### Design Risks

- **时间压力推动背板而非推理** — 如果倒计时过于压迫，玩家可能停止阅读和思考，转向记住最优路径。节奏变化是缓解手段。
- **叙事质量依赖文案水准** — 游戏的核心体验建立在文本和对话上，如果故事不吸引人，游戏就立不住。

### Technical Risks

- **循环状态管理架构** — 需要在第一行游戏代码之前设计好。将"每夜模板"和"持久变更"严格分离是关键。
- **水墨着色器质量** — 80% 正确的水墨着色器看起来 0% 艺术感。必须先于游戏逻辑进行着色器原型验证。
- **条件性对话树复杂度** — 5 住客 × 7 夜 × 跨循环状态条件 = 大量分支。推荐使用 Dialogic 或 Ink 而非自建。

### Market Risks

- **时间循环推理是利基类型** — Outer Wilds 和 Twelve Minutes 验证了受众存在，但规模有限。
- **中文独立游戏市场** — Steam 中文独立游戏竞争激烈，水墨视觉风格是差异化优势但也可能被归类为"文艺小众"。

### Scope Risks

- **内容随循环倍增** — "每夜的选择永久改变旅馆"意味着设计多个版本的同一空间。严格的 MVP 纪律至关重要。
- **着色器开发超时** — 如果着色器原型在 2-3 个专注时段内达不到质量，需要及时简化视觉风格。

### Open Questions

- **线索连接系统如何交互？** — 需要原型验证拖拽式连线 vs 对话式推理 vs 自动检测连接。原型阶段解决。
- **住客 AI 行为复杂度** — 住客是否有自己的循环内移动和时间表，还是静态等待玩家？影响开发量和推理深度。MVP 阶段先做静态。
- **多人结局如何触发？** — 3 个结局的触发条件需要在 GDD 阶段精确定义。

## MVP Definition

**Core hypothesis**: 玩家在时间压力下发现探索-推理-循环周期具有吸引力，并愿意为了"再来一次"而重复。

**Required for MVP**:
1. 3 个房间、3 个住客、3 夜循环
2. 倒计时计时器（恒定压力，节奏变化在后续迭代加入）
3. 线索笔记本（跨循环持久）
4. 基础线索连接机制（至少一种连接方式）
5. 水墨视觉风格原型（单色 + 基础色彩积累）
6. 1 个结局
7. 循环状态管理（模板 + 持久变更分离）

**Explicitly NOT in MVP** (defer to later):
- 压力节奏变化（低语/咆哮切换）
- 额外 5 个房间和 2 个住客
- 多结局
- 移动端适配
- 住客循环内移动/时间表
- 完整音效设计

### Scope Tiers

| Tier | Content | Features | Timeline |
| ---- | ---- | ---- | ---- |
| **MVP** | 3 房间、3 住客、3 夜 | 核心循环 + 线索笔记本 + 基础推理 + 恒定倒计时 | 2-3 周 |
| **Core** | 5 房间、4 住客、5 夜 | + 压力节奏 + 扩展对话 + 更深层推理 | +1-2 周 |
| **Full Vision** | 8 房间、5 住客、7 夜 | + 3 个结局 + 完整叙事 + 色彩积累完整版 | +2-3 周 |
| **Polish** | 全内容 | + 音效设计 + 视觉打磨 + 移动端适配 | +1-2 周 |
